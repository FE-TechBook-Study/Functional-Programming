## 고차 함수를 만드는 방법 #1

1. 코드를 함수로 감싸기
2. 더 일반적인 이름으로 바꾸기
3. 암묵적 인자를 드러내기
4. 함수 추출하기
5. 암묵적 인자를 드러내기

- 일급 값은 변수에 저장할 수 있고 인자로 전달 가능하며 함수의 return 값으로도 사용이 가능.
- 함수 이름에 있는 암묵적 인자는 함수의 이름으로 구분하는 코드의 냄새. 이러한 냄새를 암묵적 인자를 드러내기 리팩터링을 적용해 없앨 수 있음.
- 동작을 추상화하기 위해 본문을 콜백으로 바꾸기 리팩토링을 사용할 수 있음.

반복문 → 고차 함수로 변경

try-catch 등 → 함수 본문을 콜백으로 변경

카피-온-라이트 → 공통 함수로 빼내기

- 고차 함수로 만드는 것도 좋지만 직관적인 코드와 고차 함수를 사용한 코드중 어떤 것이 더 가독성이 좋고 명확한지 생각을 하며 코드를 작성하자.
- 어떤 코드가 더 읽기 좋고, 얼마나 중복을 많이 없앨 수 있고 다양한 관점에서 바라보며 코드를 작성하자.

## Return 하는 고차 함수

```jsx
// 리턴 받는 함수를 변수에 할당하여 사용하는 예시
function createMultiplier(multiplier) {
    // 내부 함수를 정의하여 반환
    return function(x) {
        return x * multiplier;
    };
}

// createMultiplier 함수를 호출하여 변수에 할당
const double = createMultiplier(2);
const triple = createMultiplier(3);
```

- 이렇게 만들어진 고차 함수는 변수에 할당해서 이름이 있는 일반 함수처럼 사용이 가능함
- 고차 함수를 사용하면서 중복 코드를 없애 주지만 가독성을 해칠 수도 있기 때문에 잘 익혀서 적절하게 사용해야함.
